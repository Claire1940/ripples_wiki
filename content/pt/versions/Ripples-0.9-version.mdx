<Section icon={Target} title="Como Instalar e Atualizar para a Versão 0.9 de Ripples">

A instalação da **versão 0.9 de Ripples** varia um pouco dependendo da sua plataforma de destino. Como o jogo está hospedado no itch.io e no Patreon, os jogadores devem baixar o pacote correto manualmente. Abaixo está o procedimento passo a passo oficial para garantir uma instalação limpa em plataformas de PC e dispositivos móveis.

<Steps>
  <Step num="1" title="Baixe os Arquivos Oficiais">
    Visite a página oficial no itch.io ou o perfil do Patreon do criador, Jestur. Selecione o pacote apropriado para o seu sistema operacional (Windows, macOS, Linux ou Android). Evite baixar de sites de terceiros para proteger seu dispositivo contra malwares.
  </Step>
  <Step num="2" title="Extraia o Arquivo (PC)">
    Para Windows, macOS e Linux, extraia o arquivo ZIP ou TAR baixado em uma pasta dedicada no seu disco rígido. Não execute o jogo diretamente de dentro da pasta compactada, pois isso pode causar erros nos arquivos de salvamento.
  </Step>
  <Step num="3" title="Ative Fontes Desconhecidas (Android)">
    Se estiver instalando o APK para Android, navegue até as configurações do seu dispositivo e ative "Instalar de Fontes Desconhecidas" nas opções de segurança. Abra o gerenciador de arquivos, localize o APK baixado e toque para instalar.
  </Step>
  <Step num="4" title="Verifique a Migração dos Dados de Salvamento">
    Inicie o jogo. O motor Ren'Py deve detectar automaticamente seus arquivos de salvamento existentes do Episódio 8. Se eles não aparecerem, você precisará copiá-los manualmente para o diretório de salvamento designado.
  </Step>
</Steps>

<Callout type="warning" title="Compatibilidade de Plataforma">
Certifique-se de que seu dispositivo atenda às especificações mínimas de hardware antes de iniciar o jogo. Visual novels com recursos de alta resolução podem causar gargalos de desempenho em dispositivos móveis mais antigos.
</Callout>

| Plataforma | SO Mínimo | Armazenamento Recomendado | Formato de Instalação |
| :--- | :--- | :--- | :--- |
| Windows | Windows 10 (64-bit) | 3 GB de espaço livre | Arquivo ZIP |
| macOS | macOS 11.0 ou posterior | 3 GB de espaço livre | Instalador DMG |
| Android | Android 8.0 ou posterior | 2.5 GB de espaço livre | Pacote APK |

</Section>

<Section icon={Shield} title="Solução de Problemas Técnicos e Migração de Salvamentos">

Um dos problemas mais comuns que os jogadores enfrentam ao atualizar para a **versão 0.9 de Ripples** é a perda de dados de salvamento anteriores. Como as visual novels dependem fortemente da continuidade, manter seu progresso entre as atualizações é fundamental. O jogo utiliza o motor Ren'Py, que armazena arquivos de salvamento em um local padronizado em diferentes sistemas operacionais.

Se você encontrar erros técnicos ou salvamentos ausentes, consulte a tabela abaixo para procedimentos de solução de problemas padrão:

| Problema Técnico | Causa Potencial | Solução Recomendada |
| :--- | :--- | :--- |
| Salvamentos Ausentes | Incompatibilidade de caminho de pasta | Copie os arquivos do diretório antigo do Ren'Py para o novo diretório |
| Jogo Fecha ao Iniciar | Arquivos de instalação corrompidos | Exclua a pasta do jogo e realize uma extração limpa |
| Erro de Instalação no Android | Conflito de pacotes | Desinstale a versão antiga do APK antes de instalar o APK 0.9 |
| Lentidão de Desempenho | Cache de imagens de alta resolução | Diminua o limite de cache de imagem no menu de configurações do jogo |

<Callout type="warning" title="Aviso de Backup de Salvamento">
Sempre crie uma cópia de backup do seu diretório de salvamento antes de instalar qualquer nova atualização. Uma simples falha do sistema durante o processo de atualização pode corromper permanentemente seus arquivos de progresso.
</Callout>

Para encontrar seus arquivos de salvamento manualmente, navegue pelos seguintes caminhos do sistema, dependendo do seu sistema operacional:
- **Windows**: `%APPDATA%/RenPy/Ripples-`
- **macOS**: `~/Library/RenPy/Ripples-`
- **Android**: `Android/data/com.jestur.ripples/files/saves/`

</Section>
