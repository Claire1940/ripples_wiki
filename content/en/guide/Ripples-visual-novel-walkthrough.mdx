<Section icon={Download} title="Technical Setup and Save File Porting">

Because Ripples is an episodic visual novel in active development, players must frequently update their game clients to experience new chapters like Episode 9. The game is built on the Ren'Py engine, which makes save file migration relatively straightforward, provided you follow the correct file structures. Official downloads are hosted on itch.io and supported via Patreon by the creator, Jestur.

When updating the game on desktop platforms or mobile, it is highly recommended to perform manual backups of your save directory. This prevents accidental data corruption when overwriting old game files with the latest Episode 9 build.

<Callout type="info" title="Save File Compatibility">

<Section icon={Smartphone} title="Android Installation and Troubleshooting">

Playing Ripples on Android offers a highly portable way to enjoy the visual novel, but mobile operating systems require specific permissions to handle save files and storage properly. Since the game is distributed as an APK file, you must enable installation from unknown sources in your device settings before proceeding.

If you encounter performance issues, such as slow transitions or audio desynchronization, clearing the game's cache or adjusting the text rendering speed in the in-game options menu usually resolves the problem.

<Callout type="warning" title="Android Storage Permissions">
On Android 11 and newer, you may need to manually grant storage access permissions to the Ripples application to allow it to read and write save files in your shared storage directory.
</Callout>

If you experience issues during installation or gameplay on Android, consult the troubleshooting steps below:

1. **App Not Installed Error**: This usually occurs if there is a signature conflict. Ensure you have completely uninstalled any unofficial or modified versions of the game before installing the official APK.
2. **Missing Save Files**: If your progress does not carry over, verify that the package name of the new APK matches the previous version. If the package name changed, you must manually move your files using a mobile file manager.
3. **Black Screen on Launch**: This is typically caused by insufficient RAM or corrupted assets. Try closing background applications or reinstalling the APK.
